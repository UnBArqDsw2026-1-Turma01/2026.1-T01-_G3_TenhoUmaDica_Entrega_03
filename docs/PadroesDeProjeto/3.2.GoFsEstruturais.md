# 3.2. Módulo Padrões de Projeto GoFs Estruturais


## 1. Metodologia

Para lidar com essa complexidade, aplicamos o padrão de projeto estrutural **Composite**. O objetivo deste padrão é agrupar objetos que fazem parte de uma relação "parte-todo" para tratá-los sem distinção. Com ele, a classe cliente (`BlockManager`) pode tratar tanto os blocos simples (Folhas, como `BlocoTexto` e `BlocoCodigo`) quanto os agrupamentos complexos (`BlocoLista`) de maneira uniforme, chamando apenas o método genérico `exibir()` através da interface `ComponenteBloco`, eliminando lógicas condicionais complexas de renderização.

## 3. Implementação

### 3.1 Implementação Composite (BlockManager)

Abaixo está o trecho do Cliente (`BlockManager`) instanciando a estrutura em árvore:

```typescript
// Instanciando folhas genéricas
const titulo = new BlocoTexto("Como usar Padrões de Projeto", "titulo 1");
const codigo = new BlocoCodigo("console.log('Padrão Composite rodando!');", "typescript");

// Instanciando o Composite
const postagemCompleta = new BlocoLista();

// Adicionando as partes ao todo
postagemCompleta.adicionarBloco(titulo);
postagemCompleta.adicionarBloco(codigo);

// Chamada uniforme
const resultadoHTML = postagemCompleta.exibir();
console.log(resultadoHTML);
```
